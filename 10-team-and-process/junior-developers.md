# Onboarding Junior Developers

This is something I think about for when I hire. Junior developers becoming dependent on AI before they've built fundamental skills is risky. It's like using a calculator before you understand arithmetic: you can get answers, but you don't know if they're right.

The approach that makes sense: new developers should pair with experienced engineers initially. They learn to code without AI first. Then they learn to use AI as a tool, with supervision, understanding when it helps and when it misleads.

It's slower. It's worth it.
